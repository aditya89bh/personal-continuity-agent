# Project Brief

## One-Line Thesis

A modular continuity engine that helps AI systems maintain long-term understanding of a human through memory, identity modeling, reflection, and temporal reasoning.

## Problem

Most AI assistants treat interactions as isolated sessions. They can answer well in the moment, but they struggle to maintain continuity across weeks, months, and years.

This creates failures in:

- long-term goal tracking
- personal context preservation
- behavioral pattern recognition
- identity evolution
- unfinished loop recovery
- trust and coherence across time

## Vision

Build a continuity-aware AI architecture that models not only what a user said, but how the user is evolving.

The system should support:

- structured event memory
- salience scoring
- identity-level memory extraction
- temporal reasoning
- reflection loops
- continuity recovery
- forgetting and reinforcement policies

## Initial Scope

This repository starts as a research-practice implementation, not a production assistant.

Phase 1 focuses on:

- architecture definition
- research mapping
- core memory schema
- basic salience scoring
- demo scenarios

## Target Output

A working prototype that can ingest user events, identify important memories, update a lightweight identity model, and generate continuity summaries over time.
