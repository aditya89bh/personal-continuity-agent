# Continuity Principles

## 1. Memory Is Not Continuity

Memory stores past information.

Continuity preserves meaningful change across time.

A continuity system must understand how events relate to goals, identity, behavior, commitments, and unresolved loops.

## 2. Not Everything Should Be Remembered

A useful agent must forget, compress, and prioritize.

Low-value conversational noise should decay. High-salience memories should be reinforced or reflected upon.

## 3. Identity Must Be Evidence-Backed

The identity model should not become a static personality profile.

Every belief, value, tendency, or contradiction should link back to supporting evidence.

## 4. Reflection Converts Memory Into Cognition

Raw memory is not enough.

The system needs reflection loops that synthesize events into patterns, narratives, and updates to the self-model.

## 5. Temporal Reasoning Is Central

Continuity requires tracking:

- what changed
- what repeated
- what decayed
- what stayed unresolved
- what became more important over time

## 6. Forgetting Is a Design Feature

Forgetting protects clarity, privacy, and relevance.

The system should define explicit policies for memory decay, compression, deletion, and reinforcement.

## 7. Trust Requires Inspectability

Users should be able to inspect, correct, delete, and understand memories.

A continuity agent should not become a black box biography generator.

## 8. Continuity Should Improve Action

The system is only useful if continuity improves decisions, guidance, prioritization, or recovery from context loss.

Continuity is not nostalgia. It is better orientation over time.
